You are using Log4J and/or Commons Logging in your application and all is working nicely, except for the fact that at buildtime, with Maven and CruiseControl, you always get the error log4j:WARN No appenders could be found for logger x.y.Z. What to do? You can start Maven with a system property, letting the log package know where you keep your configuration file, like so: maven -Dlog4j.configuration=src/xml/log4j.xml test (or perhaps log4j.properties, if that's your cup of tea). This works, but it's not so sweet to have to do this every time you run Maven. Well, you can supply Maven with a fixed system property by hacking maven.xml. Just add the following and you're good to go! ${systemScope.put('log4j.configuration', 'src/xml/log4j.xml')} Now things should work every time you run Maven, and you have one less problem. :)
